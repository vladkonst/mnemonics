## 1. Критические пробелы (блокируют запуск)

### 1.1 Регистрация и управление пользователями

| Компонент | Отсутствует | Последствия | Решение |
|-----------|-------------|-------------|---------|
| Команда бота | `/start` | Новые пользователи не могут начать работу | Реализовать команду с выбором роли |
| Endpoint | `POST /api/v1/users` | Невозможно создать профиль пользователя | Endpoint для создания user record |
| Endpoint | `PATCH /api/v1/users/{id}/role` | Нельзя изменить роль после регистрации | Endpoint для обновления роли |

**Критичность**: 🔴 БЛОКИРУЕТ ЗАПУСК

**Зависимости**: Все остальные функции требуют зарегистрированного пользователя

---

### 1.2 Навигация по контенту

| Компонент | Отсутствует | Последствия | Решение |
|-----------|-------------|-------------|---------|
| Команда бота | `/modules` | Студент не знает, с чего начать | Реализовать команду показа модулей |
| Endpoint | `GET /api/v1/content/modules/{id}/themes` | Нельзя увидеть список тем модуля | Endpoint для получения тем с проверкой доступа |

**Критичность**: 🔴 БЛОКИРУЕТ ОБУЧЕНИЕ

**Зависимости**: Study flow не может начаться без навигации

---

### 1.3 Система промокодов

| Компонент | Отсутствует | Последствия | Решение |
|-----------|-------------|-------------|---------|
| Endpoint | `POST /api/v1/teacher/activate-promo` | Преподаватель не может активировать код | Разделить логику от `validate-promo` |
| Endpoint | `POST /api/v1/users/join-promo` | Студент не может присоединиться по коду | Отдельный endpoint для студентов |
| Команда бота | `/join_university <code>` | Нет команды для ввода промокода | Реализовать команду |
| Поле БД | `promo_codes.expires_at` | Нельзя установить срок действия промокода | `ALTER TABLE promo_codes ADD COLUMN expires_at TIMESTAMP` |
| Поле БД | `promo_codes.status` | Нет отслеживания статуса (pending/active/expired) | `ALTER TABLE promo_codes ADD COLUMN status ENUM(...)` |

**Критичность**: 🔴 БЛОКИРУЕТ УНИВЕРСИТЕТСКИЙ ФУНКЦИОНАЛ

**Зависимости**: Весь функционал преподавателей зависит от промокодов

### 1.4 Оплата подписки

| Компонент | Отсутствует | Последствия | Решение |
|-----------|-------------|-------------|---------|
| Endpoint | `POST /api/v1/subscriptions/create-invoice` | Невозможно создать счёт на оплату | Интеграция с Payment Gateway |
| Endpoint | `POST /api/v1/webhooks/payment-callback` | Невозможно получить уведомление об оплате | Webhook handler с идемпотентностью |
| Команда бота | `/subscribe` | Нет команды для оформления подписки | Реализовать команду с выбором тарифа |
| Поле БД | `users.pending_payment_id` | Нельзя отследить ожидаемый платёж | `ALTER TABLE users ADD COLUMN pending_payment_id VARCHAR(255)` |
| Обработка | Идемпотентность webhook | Возможны дубликаты активации подписки | Проверка `SELECT subscriptions WHERE payment_id` |
| Обработка | Валидация подписи webhook | Уязвимость безопасности | Верификация подписи платёжной системы |

**Критичность**: 🔴 БЛОКИРУЕТ МОНЕТИЗАЦИЮ

**Зависимости**: Основной источник дохода

## 5. Необработанные ошибки и edge cases

### 5.1 Study Flow

| Ситуация | Текущая обработка | Проблема | Решение |
|----------|------------------|---------|---------|
| Подписка истекла между началом изучения и тестом | ❌ Нет | Студент может пройти тест после истечения подписки | Проверка подписки в `POST /user/start-test` и `submit-test` |
| Тест не пройден (score < passing_score) | ⚠️ Частичная | Неясно, можно ли пересдать без подписки | Явное поле `can_retry` в ответе |
| Нет доступа к теме | ⚠️ Частичная | Не указано, что нужно пройти | Добавить `required_theme_id` в ответ |
| Неполный набор ответов | ❌ Нет | Можно отправить 10 ответов из 20 | Валидация `len(answers) == len(questions)` |

**Критичность**: 🔴 ВЫСОКАЯ (риски безопасности и злоупотреблений)

---

### 5.2 Промокоды

| Ситуация | Текущая обработка | Проблема | Решение |
|----------|------------------|---------|---------|
| Промокод исчерпан (remaining=0) | ⚠️ Частичная | Нет информативного сообщения | Явный текст ошибки "Промокод исчерпан" |
| Промокод уже активирован студентом | ❌ Нет | Можно активировать дважды | UNIQUE constraint на (teacher_id, student_id) |
| Промокод истёк | ❌ Нет | Нет поля expires_at | Добавить поле и проверку |
| Преподаватель активирует код дважды | ❌ Нет | Перезаписывается teacher_id | Проверка `teacher_id IS NULL` |

**Критичность**: 🔴 ВЫСОКАЯ (риски злоупотреблений)

---

### 5.3 Оплата

| Ситуация | Текущая обработка | Проблема | Решение |
|----------|------------------|---------|---------|
| Платёж отклонён (payment failed) | ❌ Нет | Webhook не обрабатывает failed | Обработка status='failed' в webhook |
| Webhook приходит дважды | ❌ Нет | Возможно двойное списание | Проверка идемпотентности `SELECT WHERE payment_id` |
| Пользователь закрыл страницу оплаты | ❌ Нет | Висит pending_payment_id | Таймаут pending payment (24 часа) |
| Подписка уже активна | ❌ Нет | Можно создать второй счёт | Проверка в `create-invoice` |

**Критичность**: 🔴 КРИТИЧЕСКАЯ (финансовые риски)

---

### 5.4 Прогресс преподавателей

| Ситуация | Текущая обработка | Проблема | Решение |
|----------|------------------|---------|---------|
| Студент ещё не начал обучение | ❌ Нет | Пустой ответ | Явное сообщение "Студент не начал обучение" |
| Преподаватель пытается посмотреть чужого студента | ❌ Нет | Утечка данных | Проверка связи в teacher_promo_students |

**Критичность**: 🔴 КРИТИЧЕСКАЯ (безопасность данных)

## 6. Отсутствующие поля в БД

### 6.1 Таблица `promo_codes`

| Поле | Тип | Назначение | Приоритет |
|------|-----|-----------|-----------|
| `expires_at` | TIMESTAMP | Срок действия промокода | 🔴 ВЫСОКИЙ |
| `status` | ENUM (pending, active, expired, deactivated) | Состояние промокода | 🔴 ВЫСОКИЙ |
| `created_by_admin_id` | BIGINT | Кто создал промокод | 🟡 СРЕДНИЙ |

```sql

ALTER TABLE promo_codes ADD COLUMN created_by_admin_id BIGINT;
```

### 6.7 Новая таблица `test_attempts`

**Назначение**: История всех попыток прохождения тестов для аналитики

**Приоритет**: 🔴 ВЫСОКИЙ

**Структура**:
```sql
CREATE TABLE test_attempts (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    theme_id INT NOT NULL,
    test_id INT NOT NULL,
    answers JSONB,
    score INT NOT NULL,
    passed BOOLEAN NOT NULL,
    started_at TIMESTAMP NOT NULL,
    submitted_at TIMESTAMP NOT NULL,
    duration_seconds INT NOT NULL,

    FOREIGN KEY (user_id) REFERENCES users(telegram_id),
    FOREIGN KEY (theme_id) REFERENCES themes(id),
    FOREIGN KEY (test_id) REFERENCES tests(id),

    INDEX idx_user_theme (user_id, theme_id, submitted_at),
    INDEX idx_test_passed (test_id, passed)
);
```
